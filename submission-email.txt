## Demo Instructions

No login or credentials required. The app uses session identity (display name + room code).

**Quick demo (two browser tabs):**
1. Open https://watchtog.vercel.app in two tabs
2. Tab 1: Click "Create a Room" → name it → paste a YouTube URL like `https://youtube.com/watch?v=dQw4w9WgXcQ` → enter a display name
3. Share the 6-character room code shown in the lobby
4. Tab 2: Click "Join with Code" → enter the code → your name → you're in
5. Tab 1: Click "Enter Room" → click the **Video** button in the top bar
6. Play/pause in either tab — the other follows instantly
7. Seek to a different time — both tabs jump together
8. Send a chat message — it appears in both tabs
9. Close Tab 2 and rejoin — it syncs to the correct timestamp

## Architecture Summary

**Stack:** Next.js 14 (App Router) + TypeScript + Tailwind CSS + Framer Motion + Radix UI
**Database:** Supabase PostgreSQL with Realtime
**Realtime:** Supabase Realtime (Broadcast for sync/chat, Presence for participants)
**Video:** YouTube IFrame API (embedded) + WebRTC (screen sharing)
**Auth:** Supabase Auth — session identity (primary) + email/OAuth (optional)
**Hosting:** Vercel

**Key decisions:**
- **Dual-mode player**: YouTube embedded sync AND screen share coexist; users toggle via top bar
- **Session identity over auth**: No signup required — participant ID + display name per room
- **Ephemeral sync**: Play/pause/seek broadcast-only, never written to DB (avoids write amplification)
- **YouTube IFrame API directly**: Full control over events, error handling, and sync protocol vs a React wrapper
- **URL params + DB fallback**: Video URL passed via URL params for speed, fetched from DB on join for reliability

## Assumptions

- Each participant uses their own device and browser
- YouTube is accessible in the participant's region
- Modern browser with WebRTC/WebSocket support (Chrome, Firefox, Edge, Safari)
- Reasonably stable internet connection (>1 Mbps, <500ms latency)
- YouTube IFrame API usage complies with YouTube's terms of service
- Users are responsible for content rights when adding video URLs
- Some YouTube videos block embedding at the content owner's discretion (shown as a clear error)
- Sync is best-effort; sub-50ms typical but not guaranteed in all network conditions
- Desktop-first design; mobile responsiveness is not a current priority

---

## Known Limitations

- YouTube embedding restrictions: some content owners block embedding (clear error message shown)
- DRM-protected content (Netflix, Prime, Hotstar) cannot be screen captured by browsers
- WebRTC mesh limited to ~8 people; LiveKit SFU available as optional upgrade for larger rooms
- Chat messages are realtime only — not persisted to database
- Rooms auto-expire 12 hours after creation
- Video URL changes require users to toggle video mode to see the new video
- No playlist/watch queue support — one video at a time
- LocalChannel mode works only across same-browser tabs; cross-device requires Supabase configuration
